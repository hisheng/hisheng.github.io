p表情的mysql实战遇到的问题与解决

一.建表

建什么表，存储什么数据，以什么样的格式？

总结：

一.1.数据分为永久存储数据，还有一些为实时变动的数据，

比如user表，这个是永久存储，而不怎么经常变化的。

而某个作品的点赞的次数，这个就是变化的，你会怎么办？是存储，还是怎么办，有几个方法

1.把谁点赞谁，记一下，每当计算次数的时候，通过select查询，遍历整个表

2.专门建一个次数表，每当谁点赞的时候，在这个表，找到这个userid，然后把次数增加1

3.用redis的计数器表，即sortset类型，k-y存储次数

我的总结，mysql用来记录永久的数据，redis用来计数。在p表情上，是每当谁点赞了某个作品，我把谁，作品，记录到一个点赞表里，然后用redis记录这个作品的点赞次数增加1。这样的好处是，每当调用次数的时候，只要用k-y查询redis就可以了，然后需要知道是谁的时候，再从mysql里查询一下.

一.2.以什么样的格式呢

mysql的主要格式有

int（bigint） 整数

varchar(char,text) 字符

datetime(date,int) 时间

在p表情里，一般的id，都是用int类型，所有的json格式，以及字符用的是 varchar类型，时间方面，有datetime和int(存unix时间戳)两种类型

补充：

–unix时间戳转datetime

select from_unixtime(zp_creat_time,’%Y-%m-%d %H:%i:%s’) T from zuopin

–date类型分组

select count(‘*’) as num, date_format(user_zhuce_time,’%y-%m-%d’) from users group by date_format(user_zhuce_time,’%d’)

4.排序 select * from table_name order by ….
